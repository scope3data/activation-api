---
title: "Get Campaign Summary"
api: "GET /campaigns/{campaignId}/summary"
description: "Get natural language performance insights and health metrics for a campaign"
---

Get rich, conversational insights about campaign performance including health scores, pacing analysis, tactic breakdown, and actionable recommendations. Perfect for casual users asking "how's my campaign doing?"

## Overview

This endpoint provides **dual-mode reporting** designed for different user types:

- **üó£Ô∏è Casual Users**: Rich text summaries with insights, health scores, and ASCII charts
- **üìä Enterprise Systems**: Structured data with metrics and programmatic insights  
- **ü§ñ AI Assistants**: Natural language responses optimized for conversational interfaces

<Note>
For structured data exports suitable for BI systems, use the [Export Campaign Data](/api-reference/reporting/export-campaign-data) endpoint instead.
</Note>

## Path Parameters

<ParamField path="campaignId" type="string" required>
  The unique identifier for the campaign (format: `camp_abc123`)
</ParamField>

## Query Parameters  

<ParamField query="dateRange.start" type="string">
  Start date for analysis in YYYY-MM-DD format. Defaults to campaign start date.
</ParamField>

<ParamField query="dateRange.end" type="string">
  End date for analysis in YYYY-MM-DD format. Defaults to today.
</ParamField>

<ParamField query="includeCharts" type="boolean" default="false">
  Generate ASCII/markdown charts for visualization in text responses
</ParamField>

<ParamField query="verbosity" type="string" default="detailed">
  Summary detail level
  - `brief`: Key metrics only
  - `detailed`: Full analysis with recommendations  
  - `executive`: High-level summary for leadership
</ParamField>

Use structured fields for programmatic visualization:

```javascript

  const getHealthColor = (score) => {
    if (score >= 71) return 'green';

  };

  return (
    <div className="health-widget">
      <div className={`score ${getHealthColor(summary.healthScore)}`}>
        {summary.healthScore}/100
      </div>
      <div className="pacing">
        {summary.pacing.status === 'on_track' ? '‚úÖ' : '‚ö†Ô∏è'}

    </div>
  );
};
```

### For Automated Alerts

Check alert arrays for programmatic decision making:

```javascript
// Alert processing
const processAlerts = (summary) => {
  const criticalAlerts = summary.alerts.filter(
    alert => alert.severity === 'critical'
  );
  
  if (criticalAlerts.length > 0) {
    // Send to Slack/PagerDuty
    sendNotification({
      type: 'campaign_critical',
      campaign: summary.campaignName,
      alerts: criticalAlerts.map(a => a.message)
    });
  }
  
  // Auto-pause if severely under-delivering
  if (summary.healthScore < 30) {
    pauseCampaign(summary.campaignId);
  }
};
```

## Best Practices

<AccordionGroup>
  <Accordion title="Optimize for Your Use Case">
    - **Conversational AI**: Use `verbosity=detailed` with `includeCharts=true` 
    - **Executive Reports**: Use `verbosity=executive` for high-level insights
    - **Automated Systems**: Focus on structured fields like `healthScore` and `alerts`
    - **Frequent Monitoring**: Use `verbosity=brief` to reduce response size
  </Accordion>
  
  <Accordion title="Caching Strategy">
    Campaign summaries are computationally expensive. Consider caching:
    - **5 minutes** for active monitoring dashboards
    - **1 hour** for executive reports
    - **Daily** for historical analysis
    
    Use the `dateRange` parameter to control cache keys effectively.
  </Accordion>
  
  <Accordion title="Alert Integration">
    - Monitor `healthScore < 50` for campaign issues
    - Check `alerts` array for immediate action items
    - Use `pacing.status !== 'on_track'` for budget issues
    - Set up webhooks for real-time alerting instead of polling
  </Accordion>
  
  <Accordion title="Performance Optimization">
    - Request only the date range you need
    - Use brief verbosity for frequent requests
    - Implement client-side caching for repeated requests
    - Consider the [Export Campaign Data](/api-reference/reporting/export-campaign-data) endpoint for bulk analysis
  </Accordion>
</AccordionGroup>
