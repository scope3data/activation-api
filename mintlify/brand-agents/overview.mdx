## What is a Brand Agent?

A **Brand Agent** is the advertiser's account within the Scope3 Campaign API - the central hub that owns all the advertiser's marketing assets and campaigns. Think of it as the brand's advertising headquarters that coordinates all programmatic marketing activities.

<Note>
**Industry Standard**: Brand agents mirror the advertiser account pattern used by major platforms like Google Ads, The Trade Desk, and Facebook Business Manager.
</Note>

## Why Brand Agents Matter

### The Foundation of Everything
Brand agents are the **foundational layer** of the Scope3 API. Every resource - campaigns, creatives, audiences, and settings - belongs to a brand agent. This provides:

- **Centralized Management**: All the advertiser's marketing assets in one place
- **Resource Sharing**: Creatives and audiences reused across campaigns
- **Brand Consistency**: Unified safety rules and standards for the brand
- **Access Control**: Team permissions and API key scoping
- **Performance Analytics**: Cross-campaign insights and optimization

## Brand Agent Lifecycle

### Creation Phase
1. **Define Requirements**: Business goals, budget scale, team structure
2. **Basic Setup**: Name, domains, contact info
3. **Safety Configuration**: Brand standards and content filters
4. **Team Access**: API keys and permission levels

### Operational Phase
1. **Asset Management**: Upload and organize creatives
2. **Audience Development**: Create targeting profiles
3. **Campaign Launch**: Deploy marketing initiatives
4. **Performance Monitoring**: Track health scores and optimization

### Growth Phase
1. **Resource Sharing**: Leverage creatives across campaigns
2. **Advanced Targeting**: Refine audience definitions
3. **PMP Integration**: Access premium inventory
4. **Attribution Setup**: Connect measurement systems

## Best Practices

<AccordionGroup>
  <Accordion title="ðŸ—ï¸ Organization Strategy">
    **One Brand Agent Per Advertiser**
    - Don't create separate brand agents for different campaigns
    - Use campaign-level organization instead
    - Share resources efficiently across initiatives
    
    **Clear Naming Convention**
    - Use company or brand name as identifier
    - Add qualifier for multiple brands ("Nike Running", "Nike Basketball")
    - Keep names simple and recognizable
  </Accordion>
  
  <Accordion title="ðŸ”’ Security & Access">
    **API Key Management**
    - Generate separate keys for different teams/systems
    - Rotate keys quarterly for security
    - Never share keys in code repositories
    
    **Brand Safety First**
    - Configure comprehensive domain blocklists
    - Set up keyword filtering for sensitive topics
    - Review and update safety rules monthly
  </Accordion>
  
  <Accordion title="ðŸ“Š Performance Optimization">
    **Resource Efficiency**
    - Reuse high-performing creatives across campaigns
    - Build audience segments for different product lines
    - Share measurement configurations across initiatives
    
    **Data Strategy**
    - Set up webhook notifications for real-time monitoring
    - Configure regular data exports for BI systems
    - Use tactic analysis to optimize signal/story performance
  </Accordion>
</AccordionGroup>
